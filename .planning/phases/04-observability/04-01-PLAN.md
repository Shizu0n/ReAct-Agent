## Phase Goal

**As a** recruiter reviewing a completed run, **I want to** see how long each reasoning step took and which provider actually answered (including any fallback), **so that** the trace is legible evidence of a resilient, instrumented agent.

**This plan's slice:** the two data-generation gaps that must exist *before* traces are persisted â€” per-step `elapsed_ms` in `tool_node` (OBS-03) and redacted fallback-event capture in the provider chain (OBS-04). No database work here; this plan only makes the run produce richer, already-redacted trace data that Plan 04-02 will store and Plan 04-03 will display.

## Acceptance Criteria

- [ ] Every `Step` emitted by `tool_node` includes an integer `elapsed_ms >= 0` timing that single tool call (OBS-03).
- [ ] `FreeModelFallback` records a redacted fallback event (failed provider name + redacted error) into the `UsageTracker` when a provider fails but a later one succeeds; `summary()` exposes `fallback_events` (OBS-04).
- [ ] Fallback events carry no unredacted secret substring; the all-providers-failed `RuntimeError` behavior is unchanged (OBS-06, no-regression).
- [ ] `python -m unittest tests.test_graph tests.test_llms -v` is green; full suite has no regressions.
