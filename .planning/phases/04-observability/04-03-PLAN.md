## Phase Goal

**As a** recruiter, **I want** a run-history view in the UI where I can scan recent runs and open any one to inspect its full reasoning trace, timing, and provider fallbacks, **so that** within minutes I see legible evidence the agent is observable and resilient.

**This plan's slice:** the frontend surface for observability. A new "History" tab lists the session's runs (OBS-02); clicking one opens its persisted trace with per-step timing (OBS-03) and provider/fallback display (OBS-04); the existing About-tab eval panel is confirmed to reflect the committed baseline (OBS-05, verification only). Consumes the `/runs` and `/trace/{id}` endpoints from Plan 04-02.

## Acceptance Criteria

- [ ] A "History" nav item renders a scrollable list of the session's runs (timestamp, query summary, status, elapsed) (OBS-02).
- [ ] Clicking a run expands its full step trace with per-step `elapsed_ms` (OBS-03).
- [ ] Provider used and fallback events ("Gemini failed → Groq") are shown in the list and/or detail (OBS-04).
- [ ] Fetch is session-scoped via `X-Session-Id`; empty/error states are legible; no crash on empty history (OBS-02).
- [ ] `EvalsSection` still fetches and renders the committed baseline live on the About tab (OBS-05 verified).
- [ ] `npm run lint` and `npm run build` (`tsc -b && vite build`) both pass.
