## Phase Goal

**As a** recruiter evaluating the demo, **I want to** upload a document and ask questions grounded in it, **so that** I get cited, trustworthy answers with the retrieval visible and no hallucination when the answer is not in the document.

**This plan's slice:** the browser upload experience — a `DocumentPanel` in the chat workspace with a file picker, an uploading/progress indicator, an inline error, and a per-session document list showing filename + chunk count. This closes SC1 (visible ingestion + chunk count) and SC5 (per-session document list) end-to-end.

## Acceptance Criteria

- [ ] The chat workspace shows an upload control; picking a PDF/text file uploads it and shows an in-flight indicator (RAG-01, RAG-03).
- [ ] After success the returned chunk count is shown and the file appears in the document list (RAG-03, RAG-07).
- [ ] The document list is fetched on mount for the current session and re-fetched after each upload (RAG-07).
- [ ] Frontend `npm run build` (tsc + vite) passes.

<objective>
Add the frontend for document upload and listing without touching the agent chat flow. New types (`DocumentInfo`, `UploadResult`, `DocumentState`), a `useDocuments` hook (`uploadDocument`, `listDocuments`) that reuses the existing session id, a `DocumentPanel` component (file input + progress + inline error + list), and a mount of `<DocumentPanel sessionId={sessionId} />` inside `ChatWorkspace` (which already receives `sessionId`).

Purpose: give recruiters a legible, minutes-to-understand upload + document-list surface that proves ingestion visibly.
Output: `types/index.ts` additions, `useDocuments.ts`, `DocumentPanel.tsx`, `ChatWorkspace.tsx` mount.
</objective>

    In `frontend/src/types/index.ts` append three interfaces matching the existing style: `DocumentInfo { id: string; filename: string; chunk_count: number; created_at: string }`; `UploadResult { status: 'ok' | 'truncated' | 'empty'; filename: string; chunks_stored: number; chunks_skipped: number; doc_id: string }`; `DocumentState { documents: DocumentInfo[]; uploading: boolean; error: string | null }`.

    Create `frontend/src/hooks/useDocuments.ts` exporting a `useDocuments(sessionId: string)` hook:
    - Define a local `apiBaseUrl()` mirroring useAgent (`import.meta.env.VITE_API_URL` trimmed, else `'/api'`) — do NOT import the private one from useAgent.
    - State: `documents: DocumentInfo[]`, `uploading: boolean`, `error: string | null`.
    - `listDocuments()`: `GET ${apiBaseUrl()}/documents/${sessionId}` with header `x-session-id`; on non-ok return `[]`; parse `{ documents }` and set state; wrap in try/catch that sets `documents` to `[]` (never throws).
    - `uploadDocument(file: File)`: guard against concurrent uploads; set `uploading=true`, `error=null`; build `FormData` with `form.append('file', file)`; `POST ${apiBaseUrl()}/upload` with header `x-session-id` (do NOT set Content-Type — the browser sets the multipart boundary); on non-ok read the JSON `detail` and set `error`; on ok parse `UploadResult`, then call `listDocuments()` to refresh; finally set `uploading=false`; return the `UploadResult | null`.
    - `useEffect` on mount / when `sessionId` changes: call `listDocuments()`.
    - Read the session id from the passed-in `sessionId` prop only; do not create one.
    - Return `{ documents, uploading, error, uploadDocument }`.
  </action>
