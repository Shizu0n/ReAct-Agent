## Phase Goal

**As a** recruiter evaluating the demo, **I want to** upload a document and ask questions grounded in it, **so that** I get cited, trustworthy answers with the retrieval visible and no hallucination when the answer is not in the document.

**This plan's slice:** the HTTP surface for ingestion — a real `/upload` endpoint (with size + type guards) and a `/documents/{session_id}` list endpoint, both reachable on Vercel. After this plan a user can `curl` a PDF up and list their documents with chunk counts. The frontend widget that calls these ships in 03-03.

## Acceptance Criteria

- [ ] `POST /upload` accepts PDF/plain-text multipart, calls `ingest_document`, returns `UploadResponse` with `chunks_stored` (RAG-01, RAG-03).
- [ ] Files > 2 MB return 413; unsupported content types return 415/400 — both before ingestion (RAG-04).
- [ ] `GET /documents/{session_id}` returns per-session filenames + chunk counts (RAG-07).
- [ ] Both routes are dual-registered (bare + `/api/`) and have matching `vercel.json` rewrites; `api/index.py` `maxDuration` is 60.

<objective>
Wire the ingestion library from 03-01 into the FastAPI app: add `POST /upload` + `POST /api/upload` (multipart, 2 MB cap, content-type whitelist) and `GET /documents/{session_id}` + `GET /api/documents/{session_id}` (per-session list with chunk counts). Add the three response models. Update `vercel.json` with the two rewrites and the `functions.api/index.py.maxDuration = 60` block so large ingests do not hit the 10 s Hobby default timeout.

Purpose: make ingestion reachable over HTTP on both local and Vercel routing without changing the agent graph.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-3-04 | Denial of Service | `upload_document` | high | mitigate | 2 MB size check BEFORE read/ingest returns 413; 200-chunk cap (03-01) + maxDuration:60 bound execution (test_size_cap_returns_413) |
| T-3-05 | Tampering | `upload_document` content-type check | medium | mitigate | accept only application/pdf or text/* (or .pdf/.txt/.md) — others rejected 415 before ingestion (test_unsupported_type_returns_415) |
| T-3-03a | Information Disclosure | `list_documents` | high | mitigate | `WHERE d.session_id = %s` + `_is_valid_session_id` UUID validation; a session cannot list another session's documents |
| T-3-06 | Tampering | filename handling | low | mitigate | filename stored only in `documents.filename` TEXT column; never used as a filesystem path or written to disk |
</threat_model>
