## Phase Goal

**As a** recruiter evaluating the demo, **I want to** upload a document and ask questions grounded in it, **so that** I get cited, trustworthy answers with the retrieval visible and no hallucination when the answer is not in the document.

**This plan's slice:** the retrieval read-path — the `document_search` tool that embeds the query, does a session-scoped pgvector nearest-neighbor search, and returns cited passages behind a prompt-injection barrier, plus the system-prompt directives for citation and no-hallucination. This closes SC2 (cited answers), SC3 (visible retrieval Step), and SC4 (explicit absence, no hallucination).

## Acceptance Criteria

- [ ] `document_search` is registered in `TOOL_SCHEMAS` / `TOOL_INPUT_KEYS`, dispatched in `tool_node`, and produces a 5-key `Step` (RAG-05).
- [ ] Successful retrieval formats `[Source: filename, chunk N]` citations inside `--- BEGIN/END RETRIEVED DOCUMENTS ---` markers (RAG-06, RAG-09).
- [ ] Empty/irrelevant retrieval returns an explicit "no documents / no relevant content" observation; the system prompt forbids answering from general knowledge in that case (RAG-08).
- [ ] Retrieval is session-scoped and uses the cosine distance operator against the HNSW index; `pool` reaches `tool_node` via the `build_graph` closure (RAG-05).
