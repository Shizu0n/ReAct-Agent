| public internet → /keepalive | An unauthenticated caller can reach the endpoint and trigger a DB write |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-02 | Tampering | Unauthorized keepalive calls triggering DB writes/load | medium | mitigate | CRON_SECRET bearer-token check at the route (401 on mismatch); single-row UPDATE WHERE id=1 caps blast radius; slowapi 10/min/IP still applies |
| T-01-07 | Information Disclosure | CRON_SECRET or connection string leaking via logs on the auth path | low | mitigate | CRON_SECRET matches existing SECRET redaction marker; redaction.py already scrubs `Bearer <token>`; handler logs no raw header/URL; connection string covered by the SUPABASE marker added in Plan 02 |
| T-01-08 | Denial of Service | keepalive write path exhausting connections | low | mitigate | Uses pooler_connection() (6543), per-request context manager, single UPDATE; no connection retained |
</threat_model>
