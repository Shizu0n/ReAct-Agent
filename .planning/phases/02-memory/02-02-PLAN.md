<objective>
Add long-term memory as two visible tools — `memory_read` and `memory_write` — handled inside an async `tool_node` using the injected `AsyncPostgresStore` and the `thread_id` session namespace. Enforce the MEM-07 recency cap at write time, wrap recalled facts in a prompt-injection barrier, and steer the model (via tool descriptions + system prompt) to call them.

Purpose: This is the user-facing memory slice (SC2 cross-session fact recall, SC3 trace visibility). Because memory operations flow through `tool_node`, they get the same Step/SSE treatment as `calculator` and `web_search` with zero new trace infrastructure.

Output: An agent that stores and recalls salient facts across sessions, capped and visible.
</objective>

## Phase Goal

**As a** returning user, **I want to** have the agent recall personal facts I shared in earlier sessions ("As you mentioned before…"), **so that** the conversation feels continuous — and I can watch it read and write memory as named steps in the trace.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| recalled memory -> LLM prompt | Stored memory text re-enters the model context as a tool observation; it is untrusted user-provided content |
| API -> DB (store) | memory_read/write reach the store table via the injected AsyncPostgresStore on the pooler |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Tampering | recalled memory injected as observation | medium | mitigate | Wrap recalled facts in the `--- BEGIN/END USER MEMORIES ---` barrier (Task 1) AND instruct the model to treat that region as untrusted, not instructions (Task 2 system prompt) |
| T-02-04 | DoS | unbounded memory growth on free-tier DB | medium | mitigate | MEM-07 write-time recency cap (MAX_MEMORIES_STORED, default 20) evicts the oldest before each put; covered by MemoryCapTests |
| T-02-05 | Information Disclosure | secrets/PII written into stored memory | medium | accept + mitigate | Stored memory is NOT auto-redacted — documented in README (Task 2); do not log memory content at INFO/DEBUG (existing redaction covers logs) |
| T-02-SC | Tampering | dependencies | low | accept | No new packages; langgraph store API already installed/audited in Phase 1 |
</threat_model>
