<objective>
Lay the Phase 2 persistence plumbing: bring `api/requirements.txt` (and root `requirements.txt`) up to the upgraded LangGraph stack, add a Supabase-safe `AsyncConnectionPool` factory, wire a FastAPI lifespan that builds the checkpointer + store and exposes them on `app.state`, compile the graph with `checkpointer`/`store`, switch the agent invocation to async, extract `X-Session-Id` into a `thread_id` config, and create + run the one-time schema-setup script against the live database.

Purpose: Without this foundation every memory feature 500s in production (Vercel deploys from `api/requirements.txt`, still on `langgraph==0.2.45`) and the checkpoint/store tables do not exist on the live DB. This plan is the enabling infrastructure all later Phase 2 slices depend on.

Output: A backend that persists conversation history by session, degrades gracefully without a DB, and a live schema created via the direct connection.
</objective>
